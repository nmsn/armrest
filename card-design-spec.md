# 卡片体系设计规范 (Card Design System)

**来源**：@tou.visuals 仪表盘设计示例  
**提取日期**：2026 年 3 月  
**风格定位**：现代暗黑主题仪表盘，模块化卡片布局，强调数据可视化与视觉层次感

## 1. 整体布局原则

- **卡片网格布局**：采用不规则但视觉平衡的网格排列（类似 Bento Grid），卡片大小不一，优先将核心指标放在视觉焦点区域。
- **间距**：卡片之间保持适中间距（视觉上约 20-32px），确保呼吸感，避免拥挤。
- **背景**：纯黑背景（#000000），卡片通过不同背景色实现层次区分。
- **圆角**：所有卡片统一使用中等圆角（约 12-16px），营造柔和现代感。
- **阴影**：轻微内/外阴影（soft shadow），增强卡片浮动感和深度（避免过重阴影）。

## 2. 卡片类型与示例

### 2.1 KPI / 指标卡（Metric Card）
- **示例**：Conversion Rate（转化率）、Performance（98.9k）、Growth（8.9%）
- **结构**：
  - 顶部：小标题（灰色） + 时间范围说明（如 "In the past 1 month"）
  - 中间/右侧：大数字（突出主指标）
  - 可包含：迷你图表（sparkline / 柱状 / 点阵图）、趋势箭头（↑）
- **颜色**：主指标用亮色突出（绿色、橙色、白色），次要信息用中性灰。

### 2.2 进度 / 环形图卡（Progress / Gauge Card）
- **示例**：Activity（78%）、Skill Rate（64% 环形图）
- **结构**：
  - 标题 + 时间范围
  - 中心大数字 + 环形进度条（粗细适中，渐变效果）
  - 底部可添加图例说明

### 2.3 柱状 / 趋势图卡（Chart Card）
- **示例**：Achievement（柱状图）、Growth（紫色柱状图）
- **结构**：
  - 标题 + 趋势箭头
  - 主图表占主要空间
  - X/Y 轴标签简洁

### 2.4 列表型卡（List Card）
- **示例**：Skill Rate 右侧的技能分解列表（All Skill、UX、UI、Branding）
- **结构**：
  - 彩色点（legend） + 标签 + 百分比右对齐
  - 颜色与环形图一致

### 2.5 操作型卡（Action Card）
- **示例**：+Add Item（黑色卡片，黄色文字 + 加号图标）
- **结构**：
  - 居中大文字 + 图标
  - 强调可点击性（hover 效果建议增加亮度或缩放）

## 3. 配色规范（Color Palette）

- **背景**：纯黑 `#000000`
- **卡片背景**：
  - 白色/浅色卡：#FFFFFF（Conversion Rate、Achievement、Activity、Performance）
  - 深色卡：深绿/渐变绿（Skill Rate）、蓝色（Activity）、紫色（Growth）、纯黑（Add Item）
- **强调色**：
  - 绿色系（成功/正向指标）：亮绿、渐变绿
  - 橙色/红色：Performance 主数字
  - 紫色：Growth
  - 黄色：Add Item 操作按钮
- **文字颜色**：
  - 主标题/大数字：高对比白色或深色（依卡片背景而定）
  - 次要文字/标签：中灰（#888888 ~ #AAAAAA）
- **图表色**：绿色点阵、蓝色柱状、紫色柱状、渐变环形

## 4. 排版规范（Typography）

- **字体**：推荐现代无衬线字体（Sans-serif），如 Inter、Plus Jakarta Sans、System UI。
- **层级**：
  - **大数字（KPI）**：超大粗体（48px–72px），字重 600–700
  - **卡片标题**：14–16px，中等字重（500），灰色
  - **副标题 / 时间范围**：12–13px，轻灰
  - **图例 / 标签**：13–14px
  - **百分比**：中等大小，突出显示
- **对齐**：标题左对齐，大数字可右对齐或居中，列表右对齐百分比。

## 5. 视觉层次与可扫描性

- 每个卡片**单一焦点**：一个核心指标 + 一个辅助图表/列表。
- 使用**大小、颜色、位置**建立层次：大数字最突出，其次是图表，最后是辅助文字。
- 保持卡片**简洁**：避免在一个卡片内堆叠过多信息。
- **交互暗示**：See All 按钮、+Add Item 使用鲜明颜色；卡片整体支持 hover 效果（轻微抬升或亮度变化）。

## 6. 通用最佳实践（基于此设计）

- 卡片间保持一致的圆角和内边距（padding 约 20-24px）。
- 图表样式统一（线条粗细、颜色饱和度）。
- 时间范围说明统一放在标题下方或右上角。
- 暗黑主题下确保文字与背景对比度充足（WCAG AA 标准以上）。
- 响应式考虑：大屏使用当前网格，小屏可垂直堆叠卡片。

## 7. 实现建议（前端）

- 使用 **CSS Grid** 或 **Flexbox** 实现不规则布局。
- 圆角、阴影使用 `border-radius` + `box-shadow`。
- 环形图可使用 SVG、Canvas 或第三方库（如 Chart.js、Recharts）。
- 暗黑模式优先，配色变量化（CSS Variables）便于主题切换。
